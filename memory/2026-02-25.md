## 21:13 - 服务崩溃循环修复 ✅
- 宋老师反馈"关闭服务后没有自动重启"
- 实际情况：服务在不断重启，但每次启动 ~2s 就崩溃（exit status=1），形成崩溃循环
- 根因：systemd service 文件中 ExecStart 使用了 `--set gateway.controlUi.dangerouslyAllowHostHeaderOriginFallback=true` 参数
  - 当前版本 OpenClaw 2026.2.24 不支持 `--set` 选项（只有 `--reset`）
  - 该配置项已经存在于 `/root/.openclaw/openclaw.json` 中，命令行参数完全多余
- 修复：将 ExecStart 从带 `--set` 改为简单的 `openclaw gateway run`
- 执行 `systemctl daemon-reload && systemctl restart openclaw.service`
- 服务恢复正常运行（active/running）
- 教训：systemd service 文件中的启动命令要与当前 OpenClaw 版本的 CLI 参数保持一致
